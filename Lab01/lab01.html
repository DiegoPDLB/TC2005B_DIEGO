Preguntas del laboratorio.

¿Cuál es la diferencia entre Internet y la World Wide Web?

  • Internet: Es una red global de redes de computadoras que permite la transferencia de datos entre dispositivos. Es la infraestructura subyacente que conecta a millones de
    dispositivos en todo el mundo.
  • World Wide Web (WWW): Es un servicio específico que se ejecuta en Internet. Consiste en una colección de documentos interconectados y otros recursos que se presentan
    mediante hipervínculos y se acceden a través de un navegador web. La WWW es una aplicación de Internet que permite a los usuarios acceder y navegar por una gran cantidad
    de información en todo el mundo.

¿Cuáles son las partes de un URL?

  • Protocolo (https://)
  • Nombre o dominio
  • Ruta del recurso (Opcional el “/acceso.com”)
  • Parámetros de consulta (comienzan con “?”)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicitar datos de un recurso. No tiene efectos secundarios y se utiliza para obtener información.
  • HEAD: Similar a GET, pero solo solicita los encabezados de respuesta, no el cuerpo. Útil para obtener metadatos sin recuperar el contenido completo.
  • POST: Enviar datos al servidor para ser procesados, a menudo utilizado para enviar datos de formularios. Puede tener efectos secundarios en el servidor, como la creación de recursos.
  • PUT: Enviar datos al servidor para reemplazar completamente un recurso existente. Es idempotente y se utiliza para la actualización completa de recursos.
  • PATCH: Aplicar modificaciones parciales a un recurso existente. Similar a PUT, pero actualiza solo partes específicas del recurso.
  • DELETE: Eliminar un recurso específico del servidor

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo, cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

  • Se debe de usar el método HTTPS POST porque este mismo envía los datos en el cuerpo de la solicitud HTTP lo que los hace más seguros que el método GET, que los envía a través de la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

  • Se hace por el método GET para solicitar recursos específicos del servidor, como una página web, y devuelve el contenido solicitado al cliente (navegador).

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

  • Significa que la solicitud hecha fue exitosa y sin errores.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

  • Sí, es su responsabilidad porque al final es un error que será reportado por los usuarios del sitio.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

  • Si, es su responsabilidad porque es un error en el servidor al procesar la solicitud del usuario.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

  • Cuando un atributo o elemento HTML está "desaprobado" (deprecated) significa que aunque aún puede funcionar, se considera obsoleto y se desaconseja su uso. En HTML5, algunos elementos de HTML 4
    desaprobados son: center, font, frame, frameset, noframes, applet, isindex, entre otros. Se sugiere usar alternativas más modernas como CSS y HTML semántico.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • Nuevos elementos semánticos: HTML5 introduce elementos como header, footer, nav, article, etc., que mejoran la estructura del contenido
  • Compatibilidad multimedia mejorada: HTML5 ofrece soporte integrado para audio y video con los elementos audio y video, eliminando la necesidad de plugins externos.
  • Nuevas APIs JavaScript: HTML5 incluye APIs como geolocalización, almacenamiento local, Canvas para gráficos, Drag and Drop, etc.
  • Formularios mejorados: Introduce nuevos tipos de entrada, atributos de validación y elementos para formularios más robustos.
  • API de historial y navegación: HTML5 permite manipular el historial del navegador con la History API para crear experiencias de navegación más fluidas.
  • Soporte para almacenamiento offline: Proporciona capacidades de almacenamiento local y de aplicación web para permitir que las aplicaciones funcionen sin conexión a Internet.
  • Mejoras en accesibilidad y SEO: HTML5 incluye elementos semánticos que mejoran la accesibilidad y la indexación por parte de los motores de búsqueda.

¿Qué componentes de estructura y estilo tiene una tabla?
1.- Elementos de estructura:

  • table: Define la tabla
  • tr: Define una fila dentro de la tabla.
  • th: Define una celda de encabezado.
  • td: Define una celda de datos.
2.- Elemetos de estilo:
  • Se pueden aplicar atributos de estilo directamente a las etiquetas o utilizar CSS externo o interno para aplicar estilos avanzados, como colores de fondo, bordes, alineación, etc.

¿Cuáles son los principales controles de una forma HTML5?

  • input: Para campos de texto, contraseña, correo electrónico, número, fecha, hora, checkbox, radio, archivo, etc.
  • textarea: Para texto multilínea.
  • select: Para menús desplegables.
  • button: Para botones que pueden ejecutar acciones.
